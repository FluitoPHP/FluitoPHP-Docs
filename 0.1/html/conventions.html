Naming Conventions ↯

Talking about the naming conventions used in the framework is precisely simple camelCase and a class has the file name with suffix of .class.php where as a regular file i.e. views do not have .class before .php.

But talking about the components they always start with a capital letter for example FileManager but are not restricted where as the controllers, models and views always starts with a small letter. We will talk about all of them now.

Components ↯

The rules for naming conventions of components are not restrictive, so any naming can be used but we seriously recommend that you use CamelCase for their naming.

Modules ↯

The rules for naming conventions of modules are as restrictive as the file system of the hosted operating system and the letters should be in lower case.

Controllers ↯

Controllers on the other hand use only camelCase with the suffix of Controller, so a URL of http://<application_base_url>/<module>/XYZ/ABC will lead to the method abcAction of controller class xyzController having to extend \FluitoPHP\Controller\Controller class. Remember the URLs are case insensitive so weather the URL is in capitals or small it will not matter at all while selecting the appropriate module/controller/action.

Due to the fact that modules and controllers are the driving factor of the URLs in the application, the character in the URLs are as there can be naming conventions of the file system allowed for modules, php class naming convention and file system naming conventions for controllers and php method naming conventions for actions. Apart from these conventions only global URL conventions derive the final list of characters that can be used in the URL.

Remember: The extra URL parameters that are parsed as part of the request the case of those parameters will be untouched.

Views ↯

For the autoloading of the view the naming convention is camelCase and the file should be named as controllerAction.php. Apart from the autoload, the view names do not need to be restrictive and can be loaded freely by using the exact name/extension or the absolute path of the view name.

Models ↯

Models on the other hand shares similar class naming convention as of controllers i.e. xyzModel and the file should be named as xyzModel.class.php and they extend \FluitoPHP\Model\Model class. The extended model class can extend the methods defined in the parent model class as the developer see fit.

So now you understand the naming conventions of the different parts of the application design, you are free to start the development of your application and learn before or while the job being done.
